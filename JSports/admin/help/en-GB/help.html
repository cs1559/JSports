JSports - Sports Management

JSports is a Joomla component written specifically for Southwest Illinois Baseball League (SWIBL.org). It is used for the ongoing management of a youth baseball league; however, the component itself can be used for other sports leagues.

Component Configurations

Organization

This tab covers data attributes related to the actual organization operating the league.
  • Organization Name - Name of the organization
  • Address 1 - Mailing address (line 1)
  • Address 2 - Mailing address (line 2 if applicable)
  • City/State/Zipcode- City/State and Zipcode of the organization.
  • Contact Name = Primary contact name in the organization]
  • Email - Organizations primary email address
  • Phone - Organizations phone number

Options

  • Front End Administration
  • Registration Enabled - This setting will enable/disable registration for the entire site the component is installed on
  • Logo Path
  • Team Logo Directory Prefix
  • Remove IDs from URLs - Currently not used
  • Show Standings Position - This option will modify the standings display to include a position number such as 1, 2, 3, etc.
  • Item Id
  • CRON Key - This is a user defined key value that is used by the CRON job to update the standings to prevent unauthorized execution of the cron script.
  • Days to retain in Logs - The default number of days to keep a records in the log table.
  • Active Standings Only - This setting limits the number of programs included in the drop down display of programs to select from for displaying the standings.

Email

  • Enable Event Emails
  • FROM Email Address
  • Name of the FROM person
  • Copy League Admin
  • Admin Email(s)

Team Profile

  • Show Location - Setting to show/hide location (or city) on the team profile.
  • Show Hit Counter - Displays the number of times a profile has been viewed.
  • Show Page Links - Displays a page link menu that enables users to jump to specific sections of the profile.
  • Show Division Standings - Displays the divisional standings for the teams most recent season on the profile page.
  • Enable Team Schedule Download - Enables/Disables a users (incl. guest) ability to download a teams game schedule.
  • Enable Profile Attributes - Enables a team to manage specific attributes such as playing in tournament, making contact info public, etc.

Games

  • Reset Game Score to 0-0 - When enabled, when an admin resets a game, the score is also reset to 0-0; otherwise, the score will remain the same but the game status will be changed to scheduled.
  • Notify ADMIN of possible forfeit - This feature, when enabled, will send an email notification to the league administrators if a game entered COULD be a forfeit (7-0 score).

Permissions

League Administration

The LEAGUE is the highest level of the object hierarchy. The program can support multiple leagues. Currently, there are very few attributes related to the LEAGUE.

Program Administration

A PROGRAM is the second level of the object hierarchy. A "PROGRAM" is the same as a season. You may have a SPRING season and a FALL season. It provides some level of "time" context around the data. Below are the program configuration options.

Details

  • League- Select the league the program is associated with.
  • Description- Description of the program.
  • Sport- Sport the program is associated with (e.g. Baseball, Golf, etc.)
  • Groupings- Age groups (if applicable). ex. 7U - 14U
  • Start Date- Start Date (NOTE: only informational)
  • End Date- End Date (NOTE: only informational)

Options

  • Used for Registration only- Means the program supports registration only. Program will not support standings, teams, etc.
  • Active- Indicates of the program is active. Possible values are (A)ctive, (P)ending, (C)losed.
  • Status-
  • Publish Standings- This indicates if the standings are available on the frontend. This is used to prevent standings from displaying on the frontend while the setup/configuration of the league is being completed.
  • Setup Final- This indicates that setup of the program is FINAL and no further changes (e.g. teams move to different division) can be made.

Rosters

  • Rosters Locked- Prevents users from editing the roster on the frontend if this is set to YES.
  • Limit Roster- Indicator that forces rosters to be limited based on defined roster size.
  • Include Subtstitutes- Indicator if substitutes count toward total players on roster.
  • Roster Size- Defines the total number of players allowed on a team roster.
  • Standings/Scoring Policy-

Registration

  • Registration Open- If this option is set to YES, then this program would be available for registration in the registration flow.
  • Cost- Set the cost to register for the program/season. This will be displayed on the registratin page.
  • Introductory Text- Text displayed on the registration page before the key input elements.
  • Start Date- Date registration can begin (NOTE: Informational only).
  • End Date- Date registration ends (NOTE: Informational only)
  • Registration Template-
  • URL of Agreement-
  • Include Groups- (OPTIONAL) indicates if the registration form should include a selection for groupings (e.g. age, day, etc.)
  • Include Skills- (OPTIONAL) Presents the registrant to select skill level (e.g. Red, White, Blue)5
  • Include Playoffs- (OPTIONAL) An indicator that can be collected at registration if the team plans on participating in the playoffs
  • Include TOS Agreement- (OPTIONAL) Indicates if the person registering is requierd to accept the TOS.
  • Include Registered by- (OPTIONAL) Name of the individual who is registering a specific team.

Workflow

  • FUTURE USE. Intenion is to graphically display which what steps (e.g. registration, setup, start, end, etc.) have been completed.

Program Setup

The "program setup" is the function where regisrations/teams are assigned to different divisions within the program. This function supports a "mass input" function so assignments can be made in bulk and NOT on an individual basis. The program setup screen allows you to define a program and a group to filter on. The list of available programs is filtered to only those programs where the setup is NOT final and is not used for registration only. This both are attributes on the programs record.

Each row will present an individual drop down box. this drop down box will list all of the available divisions available for that specific program.

There is an underlying table called #__jsports_map which is updated with data entered during the setup process. This ties together the registration, team and which division the team is participating in. Each entry MUST be published for the team to appear in the standings.

Division Management

This section allows you to define divisions within a given program. League standings are all calculated within the DIVISION level. All teams must be assigned to a specific division.

Details

  • Program- Defines what program the division is associated with.
  • Age Group- Defines the specific age group for the teams assigned to the division. There are no rules to prevent any team from being placed in the wrong age/division.
  • Notes- Any additional notes/informaiton about the division. Information is currently NOT displayed on the front end. The intent is to add some additional notes such as cross-divisional play, number of games, etc. that COULD be presented on the frontend if needed.
  • Cross-Divisional Play- This flag indicates that this division, as part of the league play and setup, will be required to play teams from other divisions (within the same age group), and have those games count as part of the overall league standings.
  • Order- This is a numeric value that helps define a specific sequence when sorting the divisions.
  • Status- Published/Unpublished status. This value should not be unpublished if teams have been assigned and also if the program has already started.

Schedule

  • League Managed Schedule- This indicates it the league manages the schedule. When this indicator is set to YES, then the owner of the team profiles will NOT be able to see the 'Manage Schedule' menu option on the profile.

Teams

The Teams section contains the detailed record about the team. This is primarily data that is collected from the registration form and can be updated by team coaches/managers, who have permissions, via the team profile page.

Details

  • Contact Name- Name of the primary contact for the team. Typically, this should be the name of the head coach.
  • Contact Email- Email address of the primary contact. This attribute is HIDDEN from public display unless the profile owner explicity changes the indicator to make this public (not recommended).
  • Contact Phone- Phone number of primary contact. This attribute is HIDDEN from public display unless the profile owner explicity changes the indicator to make this public (not recommended).
  • City/State- Should represent the city of the primary contact. Could also be used to represent the City/State where the team is based out of.
  • ID- This is the team id. This is system assigned and is NOT editable.

Options

  • Profile Owner- Represents which user actually OWNS the profile. The profile owner can grant/revoke privileges who have access rights to the profile via the team roster.
  • Hits- Represents the number of times a team's profile has been viewed.

Venues

Defines all of the Venues that teams within the league play at. The venues are available for selection by users when they enter their game schedule via the team profile.

Details

  • Address- Street address of the venue.
  • City/State- City/State where the venue is located
  • ID- Unique venue id.

Location

  • Notes- General notes that may apply to the venue. The intent is to use this to specify things like number of fields or any specific instructions/notes on how to find the venue.
  • Latitute- Latitude of the venue location (for Google Maps support)
  • Longitude- Longitude of the venue location (for Google Maps support)

Registrations

This section contains the registration information from individuals who have registered a team in the league. This data is used to create teams during the league/program setup process.

Details

  • Program- Program the registration is associated with
  • Name- Head coach name
  • Address- Address of the head coach/primary contact
  • City/State- City/State of the head coach/primary contact
  • Email- Email address of the head coach/primary contact
  • Phone- Phone number of the head coach/primary contact
  • Mobile- Mobile phone of the head coach/primary contact
  • Returning Team- Indicator if the team is a returning team
  • Team Id- Unique ID of the team (if returning - otherwise it should be ZERO)
  • Age Group- Age group the team is registering for.
  • Skill Level- Skill level the team is registering for.
  • Playoffs- Indicator on whether or not the team is planning on playing in the year end playoffs.
  • Status- Published/Unpublished indicicator.

Options

  • Registered By- Name of the person who entered the registration information
  • IP Address- IP Address of the device used when registering the team
  • Acknowledged TOS- Terms of Service acknowledgement that it has been viewed and agreed to.

Games

Detailed game information.

Match

  • Program- Progrma that the game is part of.
  • Division- Division the game is associated with. In case of cross-divisional play, this division should be the division of the HOME TEAM.
  • Home Team- Home team ID.
  • Opponent- Away team ID.
  • Location- Venue location where the game is scheduled.
  • Game Date- Date of the game.
  • Game Time- Time of the game.
  • League Game- Indicator if this game is a league game and should count toward standings
  • Status- Published/Unpublished indicator.

Result

  • Home Team Score- Final score of the Home Team
  • Away Team Score- Final score of the Away Team

Rosters

Rosters contains all of the player and staff names associated with teams within the league.

Details

  • Classification- Indicates if the roster entry is a Player of Staff Member
  • First Name- First name of the player/staff member.
  • Last Name- Last name of the player/staff member.
  • Uniform Number- Uniform number of the player. DOES NOT APPLY TO STAFF MEMBERS.
  • Role- Applies to staff member. Defines what their role is.
  • User- A user account associated with this staff member. This can be used to grant admin permissions to a user who can log into the site.
  • Admin Privileges- Indicates if the user associated with the roster entry can edit the team profile.
  • Email- Staff members email address (OPTIONAL).

View Logs

The component will generate various log messages in a table called #__jsports_action_logs that can be viewed via the Adminsistrator side of Joomla. You are able to perform a search of the log messages via this backend function. The search query will look at the log message and return results based on the criteria entered.

Log files can (and should be) periodically purged to keep the underlying table as small as possible. As additional logging messages are added to the component, this table could get fairly large. Currently, the system will delete any log message 6 months or older.


Tools

This option provides a few tools that help manage the JSports installation.

  • Newsletter Import - this function will collect all emails on 'active' team rosters where an email exists as well as all registration emails currently "open" and adds it to the Coaches Mailing list. Currently, JSports supports the NSPro newsletter component only. Additional adapters are required in order for this to work with other newsletter components.
  • Purge Log Records - This function will systematically delete so many log records that are past 'n' of days. The number of days is configurable at the component level.
  • Refresh Standings - This function enables an administrator to manually refresh the league standings. This may be required if a game score is manually adjusted; however, most implementations have the standings systematially refreshed based on a time schedule (e.g. every 30 minutes).

Sponsors

This option provides the capability to manage your organizational sponsors. This includes the abilityt to add new sponsors, update existing sponsor information, add sponsorship plans to each sponsor and upload assets that campaigns can linked too in presenting sponsor information to users.

Details

  • ID- Unique Sponsor ID
  • Address Line 1- 1st line of the Sponsors address
  • Address Line 2- 2nd line of the Sponsors address (if applicable)
  • City/State/Zipcode - City/State/Zipcode of Sponsors address
  • Contact Name - Primary contact name.
  • Contact Phone - Primary contacts phone number.
  • Contact Email - Primary contacts email address.
  • Website URL - Company email.
  • Logo Filename - Name of the logo file that has been uploaded (if uploaded).
  • Slogan - One line company slogan (if applicable). NOTE: This is different than campaign text.
  • File (Logo) attachment to Upload - This is the input that enables the ability to upload a logo file. Size restrictions apply.

Sponsorships

This tab shows the existing plans that is attached to the sponsors account. Sponsorship plans can be added by clicking on the 'Add Sponsorship' button from the Sponsors EDIT screen. The following attributes are required to attacha a sponsorship plan to a sponsor:
  • Sponsorship Level - A sponsorship level is required and will define what entitlements (e.g. number of campaigns, number of bulletins, etc.) a sponsor can do on the website.
  • Type - FUTURE USE
  • Start Date - The start of the sponsorship plan.
  • End Date - The ending date of the sponsorship. This date should always be December 31 of the current year.
  • ID - The sponsorship identifier
  • Status - Publish/Unpublished status of the sponsorship.

NOTE: Program is an attribute on the sponsorship screen; however, it isn't used as the sponsorship plans are tied to a date range versus any given program.

Assets

A Sponsor may provide the organization several assets that can be used on the website as part of a given campaign. An asset can be an image or it can be a document that can be used in a campaign link. These assets are stored locally on the website. Assets can be linked to a sponsor by using the 'Add Asset' button on the Sponsor EDIT screen. The following data attributes are required when adding an asset:
  • Title - User-defined name of the asset.
  • Description - Short user-defined description of the asset.
  • Type - NOT DEFINED
  • File to Upload - An element that will open a dialog to view the local filesystem to select a file to upload.
  • Mime Type - Mime/Type of the file uploaded. This cannot be edited. This is derived during the upload process.
  • Height / Width - Height and width of the image uploaded. These values will automatically be calculated during the upload process. NOTE: This only applies to images.

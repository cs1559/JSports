League Administration

Program Administration

A PROGRAM is the second level of the object hierarchy. A "PROGRAM" is the same as a season. You may have a SPRING season and a FALL season. It provides some level of "time" context around the data. Below are the program configuration options.

Details

  • League- Select the league the program is associated with.
  • Description- Description of the program.
  • Sport- Sport the program is associated with (e.g. Baseball, Golf, etc.)
  • Groupings- Age groups (if applicable). ex. 7U - 14U
  • Start Date- Start Date (NOTE: only informational)
  • End Date- End Date (NOTE: only informational)

Options

  • Used for Registration only- Means the program supports registration only. Program will not support standings, teams, etc.
  • Active- Indicates of the program is active. Possible values are (A)ctive, (P)ending, (C)losed.
  • Status-
  • Publish Standings- This indicates if the standings are available on the frontend. This is used to prevent standings from displaying on the frontend while the setup/configuration of the league is being completed.
  • Setup Final- This indicates that setup of the program is FINAL and no further changes (e.g. teams move to different division) can be made.
  • Rosters Locked- Prevents users from editing the roster on the frontend if this is set to YES.
  • Limit Roster- Indicator that forces rosters to be limited based on defined roster size.
  • Include Subtstitutes- Indicator if substitutes count toward total players on roster.
  • Roster Size- Defines the total number of players allowed on a team roster.
  • Standings/Scoring Policy-

Registration

  • Registration Open- If this option is set to YES, then this program would be available for registration in the registration flow.
  • Cost- Set the cost to register for the program/season. This will be displayed on the registratin page.
  • Introductory Text- Text displayed on the registration page before the key input elements.
  • Start Date- Date registration can begin (NOTE: Informational only).
  • End Date- Date registration ends (NOTE: Informational only)
  • Registration Template-
  • URL of Agreement-
  • Include Groups-

  • Include Playoffs- (OPTIONAL) An indicator that can be collected at registration if the team plans on participating in the playoffs
  • Include TOS Agreement-
  • Include Registered by- (OPTIONAL) Name of the individual who is registering a specific team.

Program Setup

The "program setup" is the function where regisrations/teams are assigned to different divisions within the program. This function supports a "mass input" function so assignments can be made in bulk and NOT on an individual basis. The program setup screen allows you to define a program and a group to filter on. The list of available programs is filtered to only those programs where the setup is NOT final and is not used for registration only. This both are attributes on the programs record.

Each row will present an individual drop down box. this drop down box will list all of the available divisions available for that specific program.

There is an underlying table called #__jsports_map which is updated with data entered during the setup process. This ties together the registration, team and which division the team is participating in. Each entry MUST be published for the team to appear in the standings.

View Logs

The component will generate various log messages in a table called #__jsports_action_logs that can be viewed via the Adminsistrator side of Joomla. You are able to perform a search of the log messages via this backend function. The search query will look at the log message and return results based on the criteria entered.

Log files can (and should be) periodically purged to keep the underlying table as small as possible. As additional logging messages are added to the component, this table could get fairly large. Currently, the system will delete any log message 6 months or older.


Tools

This option provides a few tools that help manage the JSports installation.

  • Newsletter Import - this function will collect all emails on 'active' team rosters where an email exists and adds it to the Coaches Mailing list. Currently, JSports supports the NSPro newsletter component only. Additional adapters are required in order for this to work with other newsletter components.
  • Purge Log Records - This function will systematically delete so many log records that are past 'n' of days. The number of days is configurable at the component level.
